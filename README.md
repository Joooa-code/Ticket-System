# 火车票订票系统

## 项目简介

本项目是模拟 12306 火车票订票系统的后端实现，支持用户管理、车次管理、余票查询、购票、退票、候补购票、订单查询和换乘查询等功能。系统使用 C++ 编写，所有数据持久化存储在本地文件中，并利用 B+ 树作为磁盘索引，在内存受限的环境下高效运行。

## 主要功能

- **用户管理**：注册、登录、登出、查询/修改用户信息（支持权限分级 0~10）
- **车次管理**：添加、删除（未发布）、发布车次，支持多站点、多价格、多时间区间
- **余票管理**：实时维护每个车次每天的区间余票，支持并发购票和退票
- **购票与候补**：余票充足时直接购票，不足时可加入候补队列；退票时自动按 FIFO 顺序满足候补订单
- **查询功能**：查询车次详情、查询指定起止点的直达车票（可按时间或票价排序）、查询最优换乘方案（一次换乘）
- **订单管理**：用户可查看自己的所有订单（按时间倒序），并可退订指定订单

### 运行方式

程序从标准输入读取命令，每行一条，格式为：

```
[<timestamp>] <cmd> -<key1> <arg1> -<key2> <arg2> ...
```

输出结果第一行会带有 `[<timestamp>]` 前缀，后续行按命令要求输出。

## 核心设计

### 数据持久化

- 使用 `MemoryRiver` 模板类管理对象文件的读写，支持定长记录的追加、更新和随机读取。
- 使用 B+ 树（`BPlusTree`）为关键数据（用户名、车次ID、车站名、订单键等）建立磁盘索引，实现快速查找和范围查询。

### 内存管理

- 程序启动时从文件加载索引结构（B+ 树），数据对象按需从磁盘读取。
- 在线用户列表、候补队列等动态数据存储在内存中，并在修改后同步到磁盘文件。

## 数据文件

程序运行时会生成以下数据文件（均在当前工作目录）：

| 文件名 | 内容 |
|--------|------|
| `users_tree` / `users_leaf` | B+ 树索引（用户名 → 用户记录） |
| `user_count.dat` | 用户总数 |
| `train.data` | 车次静态信息 |
| `train_leaf` / `train_tree` | B+ 树索引（车次ID → 车次记录） |
| `station_leaf` / `station_tree` | B+ 树索引（车站名 → 车次位置信息） |
| `seat_data` | 余票信息 |
| `seat_leaf` / `seat_tree` | B+ 树索引（(车次ID, 日期) → 余票记录） |
| `order_data` | 订单信息 |
| `order_leaf` / `order_tree` | B+ 树索引（(用户名, 时间戳) → 订单记录） |
| `waiting_leaf` / `waiting_tree` | B+ 树索引（候补键 → 订单记录） |
